# 09a-hallmark-ui-redesign

## 目的

Hallmarkを使い、確定済みのゲーム資産と動作を維持したまま、AI生成らしく見える青紫グラデーション、ガラスパネル、均等カード、ピル表示の反復を解消します。

## 固定方針

- 初心者を含む招待利用者が迷わず対戦を完走できることを優先する
- `design.md`をアプリ全体の唯一のデザインシステムとする
- 古い星図、黒鉄、銀、卓上を専用テーマの核とする
- カード、星明りトークン、黒い星の画像を変更しない
- ゲームルール、API、認証、AWSリソースを変更しない
- UI、モーション、Webフォントの依存関係を追加しない

## 実装単位

1. [x] デザイントークンと共通状態を追加する
2. [x] ログインとロビーを非対称な台帳構造へ変更する
3. [x] 待機室を対戦票と向かい合う席として構成する
4. [x] 対戦画面を情報レール、卓上、収集台帳、操作帯へ分ける
5. [x] ルール、音量、結果表示を同じ組版へ揃える
6. [x] 320、375、414、768、1280pxで崩れないCSSへ変更する
7. [ ] 実ブラウザ2セッションでゲーム終了まで回帰確認する

## 実装結果

- 依存関係、ルート、API呼び出し、認証、ゲームルールは変更していない
- 確定済みのカード、星明りトークン、黒い星の画像資産は変更していない
- 現在フェーズと次の操作を左レールへ固定し、卓上と獲得カードを独立領域にした
- 画面幅768px以下は1列、320px相当では操作を縦積みにする
- ルールダイアログは初期フォーカス、Escape、Tab循環に対応した
- `lint`、`typecheck`、32件のテスト、`build`は成功した

実APIを使う2ユーザー回帰確認は、AWSへの変更を伴わない手動確認として別途実施します。

## 完了条件

- Hallmarkのslop testでcritical項目が残らない
- どの画面でも主要操作と現在状態を最初に判断できる
- 色の直接記述をトークンへ集約する
- クリック可能な文言が2行にならない
- 横スクロールは手札など明示した領域内だけに限定する
- lint、typecheck、test、buildが成功する
