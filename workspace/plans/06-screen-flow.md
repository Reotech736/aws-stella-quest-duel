# 06-screen-flow

## 概要

このドキュメントは、「ステラクエスト Duel」MVPの画面一覧、画面遷移、各画面の責務、PCとスマートフォン縦向きの表示方針を定義します。

`workspace/plans/03-mvp-requirements.md` と `workspace/plans/05-api-design.md` を前提とし、クライアントは表示と操作入力に集中します。ゲームルール、認可、操作可否の最終判定はサーバーが担当します。

## 設計方針

状態: 完了

- 画面ルートは認証、ホーム、待機ルーム、ゲームの4系統に絞る
- 対局結果は専用URLへ遷移せず、同じゲーム画面上へ表示する
- PCとスマートフォン縦向きを正式対応対象とする
- スマートフォンでは縦一画面を基本とし、現在必要な操作領域を画面下部へ固定する
- 機能性と状態の分かりやすさを優先し、カード、色、星、光・闇のモチーフでゲームらしさを表現する
- 色だけで情報を伝えず、文字、数字、記号、形を併用する
- APIの `availableActions` を表示制御へ使用するが、操作可否をクライアントだけで確定しない
- 通信状態が不明な間は状態変更操作を抑止し、最新状態を取得してから再開する
- タブ終了、画面遷移、ログアウトだけではルーム退出や投了を実行しない

## 画面ルート

| Route | 画面 | 認証 | 用途 |
| --- | --- | --- | --- |
| `/login` | ログイン画面 | 不要 | Cognitoへのログインと初回パスワード変更 |
| `/` | ホーム画面 | 必須 | ルーム作成、ルーム参加、復帰先判定 |
| `/rooms/{roomId}` | 待機ルーム画面 | 必須 | 参加者待ち、開始方法選択、開始前退出 |
| `/games/{gameId}` | ゲーム画面 | 必須 | 対局、投了、終了結果表示 |

公開ルーム一覧、観戦画面、対局履歴画面、ユーザー登録画面、管理者画面はMVPでは作りません。

## 起動時と認証後の共通フロー

### 起動時

1. クライアントがCognitoの認証状態を確認する
2. 未認証なら `/login` を表示する
3. 認証済みなら `GET /v1/me/context` を実行する
4. `context=null` なら `/` を表示する
5. `WAITING` または `READY` なら `/rooms/{roomId}` へ遷移する
6. `IN_GAME` なら `/games/{gameId}` へ遷移する

コンテキスト確認中はアプリ全体の読み込み表示とし、ホーム画面を一瞬表示してから別画面へ移動する挙動を避けます。

### URLを直接開いた場合

- 未認証なら `/login` へ遷移し、認証後にコンテキストを再取得する
- 認証済みでも対象ルーム・ゲームに所属していなければ、エラー通知後に `/` へ戻す
- URLのIDと `GET /v1/me/context` の所属先が異なる場合は、コンテキストの所属先を優先する
- 終了済みゲームはアクティブコンテキストがなくても、参加者本人がURLを保持していれば最終状態を取得できる

### ログアウト

- どの認証済み画面からでもログアウトできる
- ログアウト前に、通信断やログアウトではルーム退出・投了にならないことを表示する
- ログアウト操作では退出APIや投了APIを呼ばない
- Cognitoのセッションを破棄して `/login` へ遷移する

## 共通レイアウト

### ヘッダー

認証済み画面のヘッダーには次を表示します。

- アプリ名
- ログイン中の表示名
- 通信状態
- ログアウト操作

待機ルームとゲーム画面では、スマートフォンの表示領域を確保するためコンパクトなヘッダーを使用します。

### 通信状態

| 状態 | 表示 | 操作 |
| --- | --- | --- |
| 正常 | 通常表示。常時ラベルは必須としない | 許可された操作が可能 |
| 同期中 | 小さな更新表示 | 同じ状態変更操作の重複送信を無効化 |
| 再接続中 | 上部に「再接続しています」 | 状態変更操作を無効化 |
| オフライン | 上部に「通信を確認してください」 | 状態変更操作を無効化 |
| 認証切れ | ログインが必要な旨を表示 | `/login` へ遷移 |

一時的なポーリング失敗で画面全体を消さず、最後に取得できた状態を残したまま通信バナーを表示します。

### 通知

- 操作成功は、画面変化だけで十分に伝わる場合は通知を増やさない
- コピー成功、競合、通信失敗など補足が必要な結果は短い通知で表示する
- 重要な失敗は通知だけで消さず、操作領域の近くにも理由を表示する
- サーバーの `traceId` は通常表示せず、予期しないエラーの詳細表示から確認できるようにする

## ログイン画面

### 目的

管理者が作成したCognitoユーザーだけを認証します。

### 表示項目

- アプリ名
- ログイン用ユーザー名
- パスワード
- ログインボタン
- 管理者から案内されたアカウントだけが利用できる旨

### 状態

#### 通常ログイン

- 入力中のパスワードを初期状態では隠す
- 送信中は入力とボタンを無効化する
- 認証失敗時は、ユーザーの存在有無を区別しないメッセージを表示する

#### 初回パスワード変更

Cognitoから初回パスワード変更を要求された場合は、同じ `/login` 上で次を表示します。

- 新しいパスワード
- 新しいパスワード確認
- パスワード条件
- 変更してログインするボタン

変更成功後は `GET /v1/me/context` を実行し、所属先へ遷移します。

#### パスワードを忘れた場合

MVPでは利用者自身によるパスワード再発行画面を作りません。管理者へ連絡する案内を表示します。

### 対象外

- 自己登録
- SNSログイン
- 匿名ログイン
- 利用申請

## ホーム画面

### 目的

所属中のルームやゲームがないユーザーが、ルームを作成または参加します。

### 初期処理

ホーム表示前に `GET /v1/me/context` を実行します。所属が見つかった場合は、ルームまたはゲーム画面へ移動します。

### ルーム作成

- 「ルームを作成」ボタンを表示する
- 実行時に `POST /v1/rooms` を送信する
- 送信中は作成と参加の両操作を無効化する
- 成功後は `/rooms/{roomId}` へ遷移する
- 二重クリックや通信再試行では同じ `Idempotency-Key` を使用する

### ルーム参加

- 6桁ルームID入力欄
- 「ルームに参加」ボタン
- 入力は前後空白を除去し、大文字へ変換する
- 許可文字以外は入力時に説明するが、最終検証はサーバーへ委ねる
- 実行時に `POST /v1/rooms/join` を送信する
- 成功後は `/rooms/{roomId}` へ遷移する

参加失敗時は、ルームの存在、満員、期限切れ、開始済みを区別せず「ルームに参加できませんでした」と表示します。参加試行制限中も、個別ルームの状態を推測できる情報を表示しません。

### アクティブコンテキスト競合

`ACTIVE_CONTEXT_EXISTS` を受け取った場合は `GET /v1/me/context` を再実行し、既存の所属先へ移動します。

## 待機ルーム画面

### 目的

2人目の参加を待ち、作成者が開始方法を選んでゲームを開始します。

### 共通表示

- 6桁ルームID
- ルームIDをコピーするボタン
- 作成者の表示名
- 参加者の表示名、または参加待ち表示
- ルームの有効期限
- 自分の役割
- 退出ボタン

ルームIDのコピーに失敗した場合は、選択可能な文字列として表示し、手動で共有できるようにします。

### 作成者

参加者がいない場合:

- 「対戦相手の参加を待っています」と表示する
- 開始方法と開始ボタンは無効化する

参加者が揃った場合:

- 開始方法を次から1つ選択する
  - ランダム
  - 自分が先手
  - 相手が先手
- 初期選択は「ランダム」とする
- 「ゲームを開始」ボタンを表示する
- 開始中は退出、開始方法変更、開始ボタンを無効化する
- `POST /v1/rooms/{roomId}/start` 成功後に `/games/{gameId}` へ遷移する

### 参加者

- 作成者が開始するまで待つ旨を表示する
- 開始方法の選択と開始ボタンは表示しない
- ルーム状態が `IN_GAME` になったら `/games/{gameId}` へ遷移する

### ポーリング

- 5秒間隔で `GET /v1/rooms/{roomId}` を実行する
- タブ非表示時は30秒間隔にする
- 表示状態へ戻った直後に即時取得する
- `version` が同じ場合は不要な画面更新を行わない

### 退出

退出ボタンを押したら確認ダイアログを表示します。

作成者:

- ルームが閉じ、参加者も待機ルームから外れることを明示する

参加者:

- 参加枠が再開放されることを明示する

確認後に `POST /v1/rooms/{roomId}/leave` を送信し、成功後は `/` へ遷移します。

タブ終了、ブラウザバック、ログアウトでは退出APIを呼びません。

### 期限切れと状態変化

- `ROOM_EXPIRED`: 期限切れを通知して `/` へ戻す
- `ROOM_ALREADY_STARTED`: ルーム状態またはコンテキストを再取得し、ゲームへ移動する
- `VERSION_CONFLICT`: 最新ルーム状態を取得し、操作を自動再送しない
- 作成者退出によりルームが閉じた参加者: 終了理由を通知して `/` へ戻す

## ゲーム画面

### 目的

公開情報と本人だけの手札を表示し、現在のフェーズで許可された操作を入力します。

### 表示領域

ゲーム画面は次の領域で構成します。

1. ターン・フェーズ表示
2. 相手プレイヤー領域
3. 中央盤面
4. 両プレイヤーの収集状況
5. 自分のプレイヤー領域
6. 自分の手札
7. 操作領域

### ターン・フェーズ表示

常に次を短い文章で表示します。

- 現在操作するプレイヤー
- 現在のラウンドのスタートプレイヤー
- 自分が行うべき操作、または待機理由
- 黒い星マーカーの保持者

内部値をそのまま表示せず、例えば次のような文言へ変換します。

- 「あなたの手番です。カードを1枚選んでください」
- 「カードをプレイしました。追加ドローまたは手番終了を選んでください」
- 「相手がカードを選んでいます」
- 「収集する感情カードを選んでください」

### 相手プレイヤー領域

- 表示名
- 光面と闇面の星明りトークン
- 黒い星マーカー
- 手札枚数
- 各手札の色
- 収集した感情カード

相手手札には数字、カードID、複製記号を表示しません。色名または色を表す記号を必ず併記します。

### 中央盤面

- デッキの残り枚数
- デッキトップの色
- 捨て札トップ
- 現在のラウンドでプレイされたカード
- ダミーがプレイしたカード
- リードカラー
- トランプカラー。休憩カードがトップなら「トランプなし」

デッキトップには数字やカード表面を表示しません。2枚目以降の情報を推測できる表示やアニメーションも行いません。

### 自分のプレイヤー領域

- 表示名
- 光面と闇面の星明りトークン
- 黒い星マーカー
- 手札枚数
- 収集した感情カード

### 自分の手札

- 色、数字、休憩カード種別を完全表示する
- 初期表示は色別、同色内は数字昇順で並べる
- 同一カードの内部的な `a` / `b` は画面上の名称に表示しない
- プレイ可能なカードを通常表示する
- プレイ不可能なカードは選択できない見た目にする
- カードを選択しただけではAPIを送信しない
- 選択後に「このカードをプレイ」ボタンで確定する

### 操作領域

サーバーの `availableActions` に応じて、現在必要な操作だけを強調します。

| 状態 | 主操作 |
| --- | --- |
| カードプレイ前 | 追加ドロー、カード選択、カードプレイ確定 |
| カードプレイ後 | 追加ドロー、手番終了 |
| 収集カード選択 | 候補選択、収集確定 |
| 捨て札トップ選択 | 候補選択、トップ確定 |
| 相手操作待ち | 操作を表示せず待機理由を表示 |
| 終了 | 状態変更操作をすべて無効化 |

追加ドローボタンには、失う光面数と実際に引く枚数を表示します。

例:

- 「光を1失い、3枚引く」
- 手札8枚なら「光を1失い、2枚引く」
- 手札9枚なら「光を1失い、1枚引く」

手札10枚または光面1枚以下ではボタンを無効化し、実行できない理由を表示します。追加ドローは頻出操作のため確認ダイアログを挟みませんが、送信中は再実行を無効化します。

### フェーズ別表示

#### `PLAYER_TURN_BEFORE_PLAY`

自分が `currentActorPlayerId` の場合:

- `canDrawCards=true` なら追加ドローを表示する
- `playableCardIds` に含まれる手札だけ選択可能にする
- `canEndTurn=false` のため手番終了は表示しない

相手が操作中の場合:

- 相手のカードプレイ待ちを表示する
- 2秒間隔でポーリングする

#### `PLAYER_TURN_AFTER_PLAY`

自分が `currentActorPlayerId` の場合:

- プレイ済みカードを選択解除できない状態で表示する
- `canDrawCards=true` なら追加ドローを表示する
- `canEndTurn=true` なら手番終了を主操作として表示する

手番終了成功後は、ダミー、自動補充、次プレイヤーへの進行を反映したレスポンスを表示します。

#### `AWAITING_COLLECTION_CHOICE`

- 選択者本人には収集候補を強調する
- 候補選択後に「このカードを収集」ボタンで確定する
- 相手には「相手が収集カードを選んでいます」と表示する

#### `AWAITING_DISCARD_TOP_CHOICE`

- 選択者本人には捨て札トップ候補を強調する
- 候補選択後に「このカードを捨て札トップにする」ボタンで確定する
- 相手には「相手が次のトランプを選んでいます」と表示する

#### `COMPLETED` / `ABANDONED`

- ポーリングを停止する
- ゲーム操作と投了を無効化する
- 最終盤面を残して結果パネルを表示する

### ポーリング

- 相手ターンまたは相手の選択待ち: 2秒
- 自分が操作可能: 10秒
- タブ非表示: 30秒
- 表示状態へ戻った直後: 即時取得
- 操作成功後: APIレスポンス内の最新状態を即時反映
- 終了後: 停止

同じ `version` のレスポンスでは、選択中カードやスクロール位置を不要にリセットしません。

### 競合と通信失敗

`VERSION_CONFLICT`:

1. 操作を自動再送しない
2. 選択状態を解除する
3. 最新ゲーム状態を取得する
4. 「状態が更新されたため、最新の盤面を表示しました」と通知する

通信結果が不明な状態変更:

1. 同じ操作を即座に別キーで送らない
2. 同じ `Idempotency-Key` で再確認するか、最新状態を取得する
3. 状態が確認できるまで追加操作を無効化する

### 投了

- ゲーム進行中は手番やフェーズに関係なく投了操作へ到達できる
- 誤操作を避けるため、通常の主操作から視覚的に離して配置する
- 確認ダイアログで「投了すると相手の勝利となり、取り消せない」と表示する
- 確認後に `POST /v1/games/{gameId}/resign` を送信する
- 成功後は同じ画面で結果パネルを表示する

## 結果表示

結果は `/games/{gameId}` の同じ画面上に表示します。

### 表示内容

- 勝利、敗北、または勝者なし
- 終了理由
- 勝者と敗者。放棄終了では表示しない
- 最終盤面
- 両プレイヤーの最終収集カード
- 両プレイヤーの最終星明り状態
- ホームへ戻るボタン

終了理由の表示:

| `endReason` | 表示 |
| --- | --- |
| `ENLIGHTENMENT` | 1〜6の感情を集め、悟りを開いた |
| `LIGHT_LOST` | 星明りをすべて失った |
| `RESIGNATION` | プレイヤーが投了した |
| `ABANDONED` | 24時間操作がなく、勝者なしで終了した |

ホームへ戻る操作では退出や状態変更APIを呼ばず、`/` へ遷移してコンテキストを再取得します。

## PCレイアウト

### 基本配置

- 上部: ヘッダーとターン・フェーズ表示
- 上段: 相手プレイヤー、相手手札、相手収集エリア
- 中段: デッキ、捨て札、プレイエリア、ダミー
- 下段: 自分の収集エリア、自分の手札
- 右側または下部: 星明り、黒い星、現在の操作

画面幅が不足する場合は、右側の情報を下部へ移し、ゲームに必要な情報を非表示にしません。

## スマートフォン縦向きレイアウト

上から次の順に縦配置します。

1. コンパクトヘッダー
2. ターン・フェーズ表示
3. 相手プレイヤー概要
4. 中央盤面
5. 両プレイヤーの収集状況
6. 自分の状態
7. 自分の手札
8. 画面下部固定の操作領域

- ページ全体は縦スクロール可能にする
- 自分の手札だけはカードサイズを維持するため横スクロールを許可する
- 操作領域はセーフエリアを考慮し、手札や結果表示を隠さない余白を確保する
- 選択待ちでは固定操作領域の文言とボタンをフェーズに合わせて切り替える
- 横向きとタブレットはベストエフォートとする

## ダイアログ

MVPで使用する確認ダイアログ:

- 作成者のルーム退出
- 参加者のルーム退出
- 投了
- ログアウト時の注意

ダイアログは次を満たします。

- 何が起きるかを具体的に説明する
- 破壊的操作を主ボタンと同じ色だけで区別しない
- 初期フォーカスを安全なキャンセル側に置く
- Escapeキー、キャンセルボタン、背景操作の扱いを統一する
- 処理中はダイアログを閉じず、重複送信を防ぐ

## APIと画面の対応

| API | 使用画面 | 用途 |
| --- | --- | --- |
| `GET /v1/me/context` | 起動時、ホーム、復帰処理 | 所属先判定 |
| `POST /v1/rooms` | ホーム | ルーム作成 |
| `POST /v1/rooms/join` | ホーム | ルーム参加 |
| `GET /v1/rooms/{roomId}` | 待機ルーム | 待機状態ポーリング |
| `POST /v1/rooms/{roomId}/leave` | 待機ルーム | 明示退出 |
| `POST /v1/rooms/{roomId}/start` | 待機ルーム | ゲーム開始 |
| `GET /v1/games/{gameId}` | ゲーム | ゲーム状態ポーリング |
| `POST /v1/games/{gameId}/commands` | ゲーム | 追加ドロー、カードプレイ、手番終了、各選択 |
| `POST /v1/games/{gameId}/resign` | ゲーム | 投了 |

## エラー時の画面動作

| HTTP / code | 画面動作 |
| --- | --- |
| `401` | セッションを破棄し、再ログイン案内後に `/login` へ遷移 |
| `ROOM_NOT_JOINABLE` | ホームに共通の参加失敗メッセージを表示 |
| `JOIN_ATTEMPT_LIMITED` | ホームに時間をおいて再試行する旨を表示 |
| `ACTIVE_CONTEXT_EXISTS` | コンテキストを再取得して既存所属先へ遷移 |
| `ROOM_EXPIRED` | 期限切れを通知して `/` へ遷移 |
| `ROOM_NOT_FOUND` | コンテキストを再取得し、所属がなければ `/` へ遷移 |
| `GAME_NOT_FOUND` | コンテキストを再取得し、所属がなければ `/` へ遷移 |
| `VERSION_CONFLICT` | 最新状態を取得し、自動再送しない |
| `GAME_ALREADY_ENDED` | 最終状態を再取得して結果表示へ切り替える |
| `429` | 操作を抑止し、待機後に状態取得を再試行 |
| `500` / `503` | 最後の状態を保持して再接続表示 |

## アクセシビリティ

- 赤、黄、青、緑、休憩を色だけで区別せず、色名または記号を併記する
- 星明りは明暗だけでなく「光 3 / 闇 2」のように枚数を表示する
- 選択中、選択可能、選択不可を枠線、記号、文言でも区別する
- PCではカードと主要操作をキーボードで選択できるようにする
- フォーカス位置を常に視認可能にする
- ダイアログ表示中は背面へフォーカスを移動させない
- タッチ対象は隣接操作との誤操作を避けられる大きさと間隔を確保する
- 自動更新で読み上げを過剰に発生させず、手番変更や重要エラーだけを通知対象にする
- 動きの軽減設定を尊重し、ゲーム進行に必須のアニメーションを作らない

## 画面設計の検証シナリオ

- 未認証ユーザーが保護画面からログインへ移動する
- 初回パスワード変更後に正しい所属先へ移動する
- 認証済みで所属がないユーザーがホームを表示できる
- ルーム作成後、待機ルームへ移動できる
- 無効なルームIDの詳細を明らかにせず参加失敗を表示できる
- 参加者が入室すると作成者だけに開始操作が表示される
- 作成者が開始方法を選び、両者が同じゲームへ移動する
- 作成者退出と参加者退出で異なる確認内容を表示する
- 再読み込み後に待機ルームまたは進行中ゲームへ復帰できる
- 自分の手札は数字まで表示し、相手手札は色だけを表示する
- デッキトップは色だけを表示する
- プレイ不可能なカードを選択できない
- カードプレイ後に追加ドローと手番終了を選べる
- 手札8枚、9枚、10枚で追加ドロー表示が正しく変わる
- 収集カードと捨て札トップの候補だけを選択できる
- 相手操作中は待機理由を表示し、操作ボタンを出さない
- 競合時に操作を自動再送せず最新盤面へ更新する
- 通信状態が不明な間に重複操作できない
- 手番外でも投了確認へ到達できる
- 終了時に同じゲーム画面で最終盤面と結果を確認できる
- 放棄終了で勝者と敗者を表示しない
- PCとスマートフォン縦向きで必要情報と操作へ到達できる
- 色覚に依存せずカード色、星明り、操作状態を判別できる
